# 红队培训考题 · DH 流量还原 Writeup

## 0. 题目信息

- 流量文件：`out/out/out`（疑似 `SCADA/设备` 通信抓包，文本形式的十六进制 token 流）
- 加密脚本：`out/out/task.py`
- 目标：从流量中恢复被加密的 `flag`

## 1. 加密逻辑复盘（task.py）

```python
g = 916143391925527262831875920931
p = 14396524142538228424993723224595141948383030778566133225922417832357880258148761185020930195532450742879746914027266864394266451377581759004827248578768524336431104

def gen_pub(x):           return pow(g, x, p)              # 公钥
def gen_sharekey(A, b):   return sha256(str(pow(A, b, p)))[:16]   # 共享密钥(前16字节)
def pad(msg):             return msg + b'\x00' * (16 - len(msg) % 16)   # \x00 填充
def encrypt(key, msg):    return AES.new(key, AES.MODE_ECB).encrypt(pad(msg))  # AES-ECB
```

流程是典型的 **Diffie-Hellman + AES-ECB**：
1. 主站随机指数 `a`，公钥 `A = g^a mod p` 发给子站；
2. 子站随机指数 `b`，公钥 `B = g^b mod p` 发给主站；
3. 双方各自算共享密钥 `K = sha256(str(g^{ab} mod p))[:16]`；
4. 主站用 `K` 加密 flag → `C1`（48 字节，3 个 AES 块）；子站用 `K` 加密回执 → `C2`（16 字节）。

若 `p` 是安全大素数，这就是标准 DH，无法离线破解。但本题的 `p` 有致命缺陷。

## 2. 致命弱点：p = 2^542 是合数

```python
>>> import sympy
>>> N = p.bit_length() - 1
>>> p == (1 << N)
True
>>> sympy.isprime(p)
False
```

`p` 根本不是素数，而是 **2 的幂 `2^542`**。这意味着：

- 模运算发生在环 `Z_{2^542}` 上，而非有限域；
- `g` 不是原根（因为根本不存在原根），DH 的“离散对数难”假设在这里**完全不成立**；
- 群 `Z_{2^n}^* (n>=3)` 的结构是已知的：`Z_{2^n}^* ≅ <-1> × <5>`，阶为 `2^{n-1}`，其中 `<5>` 是 `≡1 (mod 4)` 的子群，阶 `2^{n-2}`。

> 这就是题目命门：把 DH 建在 `2` 的幂模数上，离散对数变成**可计算的 2-adic（p-adic）问题**。

## 3. 解题方法：2-adic 离散对数

目标：由 `A = g^a mod 2^N` 反推 `a`（同理 `B = g^b` 反推 `b`）。

### 3.1 把元素分解到 `<-1> × <5>` 基

任意奇数 `h` 唯一写成 `h = (-1)^e · 5^f`，其中：
- `e = 0` 若 `h ≡ 1 (mod 4)`，否则 `e = 1`（即 `-1` 分量）；
- `v = h`（当 `e=0`）或 `v = -h mod 2^N`（当 `e=1`），此时 `v ≡ 1 (mod 4)`，可落到 `<5>` 子群。

于是对 `g` 与 `A` 分别得到 `(e_g, f_g)` 与 `(e_A, f_A)`，满足：
```
A = g^a  ⟹  (-1)^{e_g·a}·5^{f_g·a} = (-1)^{e_A}·5^{f_A}
```
得到两个同余：
```
e_g · a ≡ e_A   (mod 2)
f_g · a ≡ f_A   (mod 2^{N-2})
```

### 3.2 求 f = dlog_5(v)（5 基 p-adic 对数）

用 Hensel 逐位提升，求 `f` 使 `5^f ≡ v (mod 2^N)`，已知 `v ≡ 1 (mod 4)`：

- 不变量：`5^f ≡ v (mod 2^j)`。
- `j` 从 2 递增；考察第 `(j-2)` 位，候选 `f' = f + 2^{j-2}`。
- 关键恒等式：`5^{2^{j-2}} ≡ 1 + 2^j (mod 2^{j+1})`。
- 因此 `5^{f'} ≡ v·(1+2^j) ≡ v + v·2^j (mod 2^{j+1})`。
- 当且仅当 `5^f ≡ v + 2^j (mod 2^{j+1})`（即当前 `f` 在 `2^{j+1}` 精度下不对）时，`5^{f'} ≡ v (mod 2^{j+1})`，**此时才置位**。

> ⚠️ 初版脚本的 bug：判定条件写成 `!=`（提升结果“不等于 v”就置位），方向反了，导致逐位全部误判、收敛到 `2^{N-2}` 附近的垃圾值。正确条件是**“等于 v”才置位**。

```python
def dlog5(v, n):
    assert v % 4 == 1
    f = 0
    for j in range(2, n):
        if pow(5, f + (1 << (j - 2)), 1 << n) % (1 << (j + 1)) == v % (1 << (j + 1)):
            f += (1 << (j - 2))
    return f
```

### 3.3 合并同余得到 a

- `t = v2(f_g)`（即 `f_g` 最低置位），把 `f_g·a ≡ f_A (mod 2^{N-2})` 两边除以 `2^t`，模降为 `2^{N-2-t}`，用奇数逆元解出 `a ≡ a0 (mod 2^{N-2-t})`。
- 再叠加 `e_g·a ≡ e_A (mod 2)` 的奇偶约束，枚举得到所有候选 `a`（2-adic 下多值，本题实际唯一）。
- 对 `B` 同样解得候选 `b`。

### 3.4 还原共享密钥并解密

```python
shared = pow(B, a, p)          # = g^{ab} mod p，与 pow(A, b, p) 一致
key = sha256(str(shared)).hexdigest()[:16].encode()
pt  = AES.new(key, AES.MODE_ECB).decrypt(C1)   # 去 \x00 填充
```

> 稳健性处理：因 2-adic 对数存在 coset 多值，脚本枚举所有 `(a, b)` 候选组合，取满足 `pow(B,a)==pow(A,b)` 且解密明文呈 `flag{...}` 的那组，避免“解对了 DLP 但选错 coset 导致密钥错”的陷阱。

## 4. 流量解析要点

流量是“方向: 十六进制字节流”的文本行，包结构：
- 第 6 字节 `b[6]` 为类型：`1` = DH 公钥（ASCII 十六进制字符串，Python2 `long` 带尾随 `L`）；`2` = 密文（原始字节）；
- `b[4]*256 + b[5]` 为 seq；`b[7:]` 为 payload。

关键坑：字符串 `"主站" in "子站->主站"` 为真，所以必须用**精确方向串** `'主站->子站'` / `'子站->主站'` 判定，否则主/子站公钥、密文会错位。

解析后：
- `主站->子站` type1 → 公钥 `A`（450 bit）；
- `子站->主站` type1 → 公钥 `B`（451 bit）；
- `主站->子站` type2 → 密文 `C1`（48 B）；
- `子站->主站` type2 → 密文 `C2`（16 B）。

## 7. 复盘 & 加固建议

- **不要**把 DH 建在 `2` 的幂或任何合数模数上：此时群的离散对数可高效求解（2-adic 提升是经典手法）。
- 模数必须是**强素数**或满足 DLP 困难性的大素数（如 2048+ 位的 safe prime），并配 DH 参数校验（拒绝可退化模数）。
- 流量中明文传输公钥与密文、且无完整性校验，实战中应加 TLS / 验签。
